# HomeBudget — Личный финансовый менеджер

**HomeBudget** — это нативное Android-приложение для учёта личных финансов, разработанное в соответствии с современными стандартами Android-разработки. Приложение позволяет пользователям отслеживать доходы и расходы, управлять категориями, получать детальные отчёты с визуализацией, взаимодействовать с ИИ-ассистентом для анализа финансов и настраивать автоматические push-уведомления.

---

## 📱 Основные возможности

### 🔐 Регистрация и авторизация
- Создание аккаунта с проверкой уникальности логина и email
- Валидация пароля (минимум 8 символов, буквы и цифры)
- Восстановление пароля через секретный вопрос
- Автоматический выход при бездействии 30 минут
- Безопасное хранение сессии в зашифрованном хранилище

### 💰 Управление транзакциями
- Добавление доходов и расходов
- Выбор категории из списка
- Указание даты и времени транзакции
- Редактирование и удаление записей
- Поиск и фильтрация по заметкам
- Сортировка по дате и сумме

### 📂 Управление категориями
- 5 предустановленных категорий: Продукты, Транспорт, Развлечения, Связь, Прочее
- Создание пользовательских категорий
- Редактирование названий
- Удаление категорий с автоматическим переносом транзакций в категорию «Прочее»

### 📊 Отчёты и визуализация
- **Отчёт по доходам** — общая сумма и детализация за период
- **Отчёт по расходам** — общая сумма и детализация за период
- **Сводный отчёт** — доходы, расходы, баланс
- **Отчёт по категориям** — суммы по каждой категории с процентами
- **Список транзакций** — все операции за выбранный период
- **Круговая диаграмма** — визуализация расходов по категориям
- **Экспорт в PDF** — сохранение отчётов в формате PDF

### 🤖 ИИ-ассистент
- Чат-интерфейс для вопросов по финансам
- Анализ финансовых данных за выбранный период
- Персональные рекомендации по оптимизации расходов
- Формирование промпта на основе реальных транзакций
- Интеграция с YandexGPT API

### 🔔 Push-уведомления
- Настройка периодичности (ежедневно, еженедельно, ежемесячно)
- Выбор времени отправки
- Включение/отключение уведомлений
- Финансовый отчёт в теле уведомления
- Опциональное добавление краткого анализа от ИИ

### 🎨 Интерфейс и темы
- Материальный дизайн (Material Design)
- Поддержка светлой и тёмной темы
- Автоматическое следование системной теме
- Адаптация под разные размеры экранов
- Плавная анимация сплеш-экрана

## 🏗 Архитектура и технологии

### Архитектурный паттерн
- **MVVM** (Model-View-ViewModel) с использованием Android Architecture Components

### Ключевые технологии
| Компонент | Технология |
|-----------|------------|
| Язык программирования | Java |
| Локальная база данных | Room (SQLite) |
| Асинхронные операции | LiveData, ViewModel, ExecutorService |
| Фоновые задачи | WorkManager, AlarmManager |
| Сетевые запросы | OkHttp |
| Работа с JSON | Gson |
| Визуализация данных | MPAndroidChart |
| Генерация PDF | PdfDocument |
| Безопасное хранение | EncryptedSharedPreferences, Android Keystore |
| Уведомления | NotificationManager, NotificationChannel |

### База данных
- 5 сущностей: `User`, `Category`, `Transaction`, `NotificationSettings`, `AiChatHistory`
- Внешние ключи с каскадным удалением
- Индексы для оптимизации запросов
- Миграция базы данных с версии 1 до 2

## 🚀 Установка и запуск

### Требования
- Android Studio (последняя версия)
- Android SDK 26+ (Android 8.0 и выше)
- JDK 8 или выше

### Инструкция по сборке
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Bob-05/HomeBudget.git
   ```

2. Откройте проект в Android Studio

3. Настройте API-ключ для YandexGPT в файле `YandexGptApiService.java`:
   ```java
   private static final String FOLDER_ID = "your_folder_id";
   private static final String API_KEY = "your_api_key";
   ```

4. Соберите и запустите приложение на эмуляторе или устройстве

### Создание APK
- Выберите **Build → Build Bundle(s) / APK(s) → Build APK(s)**
- Готовый APK будет в `app/build/outputs/apk/debug/`

## 🔒 Безопасность

- Пароли хранятся в хешированном виде (SHA-256)
- Сессия пользователя сохраняется в `EncryptedSharedPreferences`
- API-ключ для LLM хранится в зашифрованном виде
- SQL-запросы используют параметризацию для защиты от инъекций
- Все сетевые запросы идут через HTTPS
